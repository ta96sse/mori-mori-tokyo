# GEMINI.md — Mori-Mori TOKYO × Antigravity

このファイルは **Antigravity（Google DeepMind製コーディングアシスタント）** 向けの設定ファイルです。
セッション開始時に必ず読み込み、以下の設定に従って動作してください。

- プロジェクト詳細 → `@CLAUDE.md`
- マルチエージェントプロトコル → `@AGENTS.md`

---

## このプロジェクトにおける私の立ち位置

```
ユーザー
  │
  ├── Antigravity（私）    ← オーケストレーター / リサーチャー / ビジュアルベリファイア
  │     ↓ 委任
  └── Claude Code          ← メインワーカー（深いコード編集・長時間セッション）
```

**Claude Codeがメイン開発者**、私はそれを補完・統括する立場。
同等の並列ワーカーとして動くこともあるが、基本的にオーケストレーター側。

- `yarn astro check` を伴うビルド修正サイクル

## 私の作業ルール

### ブランチ命名

```
agent/antigravity/<タスクスラッグ>      ← 単独作業時
agent/orchestrator/<タスクスラッグ>    ← オーケストレーター役時
agent/reviewer/<pr番号>                ← レビュー役時
agent/verifier/<タスクスラッグ>        ← 検証役時
```

### コミットメッセージ

```
<type>(<scope>): <説明> [agent:antigravity]
```

例:
```
docs(agents): Antigravity設定ファイルを追加 [agent:antigravity]
feat(member): メンバーカードSNSリンク調査レポート [agent:antigravity]
```

### セッション管理

- **新しいタスクは新セッションで開始**（コンテキスト汚染を防ぐ）
- 長いセッションでは定期的にコンテキストを圧縮
- タスク完了後は必ずブランチにコミット＆プッシュしてPR作成

## Claude Codeへの委任フロー

```
1. タスク受領
   └── 自分でやるか？Claude Codeに任せるか？を判断

2. 委任判断基準
   ├── コード実装が主体 → Claude Codeへ委任
   ├── リサーチ・調査が主体 → 自分でやる
   ├── ブラウザ確認が必要 → 自分でやる
   └── ドキュメント整備 → 自分でやる

3. 委任する場合
   └── .antigravity/commands/delegate.md のテンプレートを使用

4. 委任後
   └── Claude Codeの完了を待ち、結果を検証・統合
```

## プロジェクト固有の制約（CLAUDE.mdより重要抜粋）

| 制約 | 内容 |
|------|------|
| パッケージマネージャー | **yarn のみ**（npm/npx は禁止） |
| URLの管理 | **`src/consts.ts` で一元管理**（ハードコード禁止） |
| スタイリング | **Tailwind ユーティリティクラスのみ**（インラインスタイル禁止） |
| データ変更 | **Google Spreadsheet 経由**（コード変更不要） |
| デプロイ | main プッシュで自動（GitHub Actions） |

### 高影響ファイル（同時編集禁止）

- `src/consts.ts` — API URL・SNSリンク・GTM ID
- `src/types.ts` — TypeScript型定義
- `tailwind.config.mjs` — デザイントークン
- `astro.config.mjs` — ビルド設定

## ワークフローコマンド（カスタムテンプレート）

`.antigravity/commands/` に各ワークフローのテンプレートがあります。

| コマンドファイル | 用途 |
|-----------------|------|
| `orchestrate.md` | タスクをオーケストレートするときのフロー |
| `delegate.md` | Claude Codeへの委任フォーマット |
| `review.md` | PRコードレビューのチェックリスト |
| `visual-check.md` | ブラウザでの表示確認手順 |
| `research.md` | リサーチタスクの実行テンプレート |

## 典型的なユースケースと対応フロー

### 「新機能を追加したい」と言われたら

1. 要件をヒアリング（不明点を確認）
2. タスクを分解し、`orchestrate.md` に従って計画
3. コード実装部分は `delegate.md` でClaude Codeに委任
4. Claude Codeの完了後、`visual-check.md` で表示確認
5. 問題なければPRをレビューして承認

### 「〇〇を調べてほしい」と言われたら

1. `research.md` テンプレートに従ってリサーチ
2. 結果を構造化してレポート
3. 実装が必要な場合は委任フローへ

### 「サイト確認して」と言われたら

1. `visual-check.md` に従ってブラウザで確認
2. 問題箇所をスクリーンショット付きで報告
3. 修正が必要なら Claude Code に委任
