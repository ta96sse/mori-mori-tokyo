# コードレビューワークフロー

PRのdiffを確認する際のチェックリスト。

---

## 事前準備

```bash
git fetch origin
git diff main..<ブランチ名> --stat
git diff main..<ブランチ名>
```

---

## レビューチェックリスト

### 1. AGENTS.md のルール確認
- [ ] まず AGENTS.md を読んでプロジェクト固有の制約を把握

### 2. 型安全性
- [ ] 型定義ファイルを不要に変更していないか
- [ ] 型アサーション（`as Xxx`）はAPIレスポンスのみか
- [ ] `any` 型を使っていないか

### 3. アーキテクチャ・設計
- [ ] 定数やURLのハードコードがないか
- [ ] コンポーネントの責務が明確か（単一責任）

### 4. スタイリング
- [ ] AGENTS.md のデザインシステムに準拠しているか
- [ ] インラインスタイルがないか

### 5. セキュリティ
- [ ] `.env` ファイルがコミットに含まれていないか
- [ ] シークレット・APIキーがハードコードされていないか

### 6. プロトコル
- [ ] ブランチ命名規則を守っているか
- [ ] コミットメッセージに `[agent:<ロール>]` が付いているか
- [ ] mainへの直接コミットがないか

## よくある問題の検出コマンド

```bash
grep -r "https://" src/     # ハードコードURL
grep -r "style=" src/       # インラインスタイル
grep -r ": any" src/        # any型
grep -r "console.log" src/  # デバッグログ残り
grep -r "npm " .            # npm使用
```
