# リサーチタスクテンプレート

ライブラリ調査・ベスプラ調査・技術選定などのリサーチタスク用。

---

## リサーチの種類

| 種類 | 使うツール | 例 |
|------|-----------|-----|
| 最新情報・ベスプラ | `search_web` | 「Astro 5の新機能」「Tailwind CSS v4の変更点」 |
| ライブラリ比較 | `search_web` + `read_url_content` | 「アニメーションライブラリの比較」 |
| API仕様確認 | `read_url_content` | 「GAS APIのドキュメント確認」 |
| GitHubリポジトリ調査 | `read_url_content` | 「参考実装のソースを確認」 |
| 本番サイト確認 | `browser_subagent` | 「他チームのサイトを参考確認」 |

---

## リサーチ実行テンプレート

### STEP 1: リサーチ設計

```
リサーチテーマ: [何を調べるか]
目的: [なぜ調べるか・何の判断に使うか]
アウトプット: [結果をどう使うか]

調査観点:
1. [観点1]
2. [観点2]
3. [観点3]

制約・前提:
制約・前提:
- AGENTS.md の制約事項・技術スタックを参照
- 変更できないもの: AGENTS.md の高影響ファイルセクションを参照
```

### STEP 2: 情報収集

`search_web` クエリ例:
```
[ライブラリ名] best practices 2025
[技術名] vs [技術名] comparison [year]
[ライブラリ名] astro integration example
[機能名] tailwind css implementation
```

### STEP 3: 検証・クロスリファレンス

- 複数ソースで情報を確認
- 公式ドキュメントを最優先
- 実装例のコードが現在のプロジェクト制約と整合するか確認

## リサーチレポートフォーマット

```markdown
## リサーチレポート: [テーマ]

**調査日**: [YYYY-MM-DD]
**調査者**: Antigravity [agent:antigravity]

### TL;DR（結論・推奨）

[1〜3行で結論を述べる]

### 調査結果

#### [観点1]
- [発見1]
- [発見2]

#### [観点2]
- [発見1]
- [発見2]

### このプロジェクトへの適用可能性

| 項目 | 評価 | 理由 |
|------|------|------|
| [ライブラリA] | ✅ 採用推奨 | [理由] |
| [ライブラリB] | ❌ 非推奨 | [理由] |

### 次のアクション

- [ ] [実装するなら何をするか]
- [ ] [確認が必要な点]

### 参考リンク

- [URL1]
- [URL2]
```

## このプロジェクトで調べるべき定期チェック項目

四半期ごとに以下を確認することを推奨:

- [ ] Astro の最新バージョン・マイグレーションガイド
- [ ] Tailwind CSS の更新情報
- [ ] GitHub Actions の `actions/checkout` 等の更新
- [ ] GAS API の仕様変更
- [ ] Google Fonts の新しい日本語フォント（`font-jp`, `font-serif` の代替候補）
