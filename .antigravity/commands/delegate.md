# タスク委任テンプレート

他のエージェント（Claude Code等）へタスクを委任する際のフォーマット。

---

## 委任JSON（AGENTS.md 準拠）

```json
{
  "task_id": "[YYYYMMDD-タスクスラッグ]",
  "role": "worker",
  "input": {
    "description": "[タスクの詳細説明]",
    "target_files": ["[対象ファイルパス]"],
    "constraints": [
      "AGENTS.md の制約事項を遵守すること"
    ],
    "context_files": ["AGENTS.md", "[関連ファイル]"]
  },
  "output_format": {
    "type": "git_branch",
    "branch": "agent/worker/[YYYYMMDD-タスクスラッグ]"
  }
}
```

---

## 委任チェックリスト

- [ ] タスクの説明は具体的か（何を、どのファイルに、どう実装するか）
- [ ] 完了条件は明記したか
- [ ] 参照すべきファイルは `context_files` に含めたか
- [ ] 高影響ファイルへの変更を含む場合、ドラフトPRを先に作成したか
- [ ] ブランチ名はルール通りか（`agent/<ロール>/<スラッグ>`）

---

## 委任の判断基準

✅ 委任すべき:
- コンポーネントの実装・大幅な変更
- 複数ファイルにまたがるリファクタリング
- ビルドエラーの修正ループ
- CSS・スタイルの細かい調整

❌ 委任しない（自分でやる）:
- ブラウザでの表示確認
- ライブラリ・API の調査
- ドキュメントの更新
- タスクの設計・分解

---

## 緊急委任（ショートフォーマット）

```
ブランチ: agent/worker/[スラッグ]
タスク: [1〜2行で説明]
制約: AGENTS.md 参照
参照: AGENTS.md, [関連ファイル]
完了後: PR作成
```
