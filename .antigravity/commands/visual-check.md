# ブラウザ表示確認ワークフロー

`browser_subagent` ツールを使ったビジュアル検証の手順。

---

## 確認対象URL

| 環境 | URL |
|------|-----|
| 本番 | https://mori-mori.tokyo |
| ローカル（yarn dev起動後） | http://localhost:4321 |
| GitHub Pages プレビュー | PR のデプロイプレビューURL（あれば） |

---

## 標準チェックリスト

### 1. トップ / ナビゲーション

- [ ] ロゴが正しく表示されているか
- [ ] ナビゲーションリンクが機能しているか
- [ ] ヒーローセクションのコピーが正しいか
- [ ] 背景色が `#050505`（bg-dark）になっているか

### 2. ABOUT セクション

- [ ] テキストが正しく表示されているか
- [ ] アニメーション（reveal）が動作しているか

### 3. NEXT セクション（次回レース）

- [ ] 次回レースが正しく表示されているか
- [ ] 日付・場所・距離が正しいか
- [ ] データがない場合の表示が適切か

### 4. LOG セクション（過去レース）

- [ ] 過去レース一覧が正しく表示されているか
- [ ] `show_in_log: "TRUE"` のレースのみ表示されているか

### 5. MEMBER セクション

- [ ] メンバーカードが全員分表示されているか
- [ ] SNSリンクが正しく機能しているか
- [ ] アバター画像が表示されているか

### 6. レスポンシブ確認

- [ ] モバイル（375px）で崩れていないか
- [ ] タブレット（768px）で崩れていないか
- [ ] デスクトップ（1280px）で崩れていないか

### 7. パフォーマンス・アクセシビリティ

- [ ] 画像が表示されているか（broken imageがないか）
- [ ] フォントが正しく読み込まれているか
- [ ] 主要なリンクのhrefが空でないか

## browser_subagent への指示テンプレート

```
以下のURLをブラウザで開いて確認してください: [URL]

確認項目:
1. [確認したい項目1]
2. [確認したい項目2]
3. [確認したい項目3]

各確認項目について「OK / NG + 詳細」形式でレポートしてください。
NGの場合はスクリーンショットを取得してください。

完了条件: 全確認項目のレポートが完成したら終了してください。
```

---

## 問題発見時のフロー

```
問題発見
  ↓
スクリーンショットで記録
  ↓
問題を分類:
  ├── データの問題 → Google Spreadsheetを確認
  ├── CSSの問題 → Claude Codeへ委任（delegate.md）
  ├── コンポーネントの問題 → Claude Codeへ委任
  └── ビルドの問題 → yarn astro check を実行して確認
```
